# Agent Templates

`kforge` can install deterministic instruction files for common LLM coding
agents. These files tell an agent how to operate inside a knowledge repo without
changing the protocol or depending on a model provider.

The templates are plain Markdown generated by the TypeScript CLI. They do not
call a model, create a vector database, require Python, or use the network.

## List Templates

```bash
kforge agent list
```

Available templates:

| Template | Installed file | Intended client |
| --- | --- | --- |
| `agents` | `AGENTS.md` | Codex, OpenAI agents, and general repo-aware coding agents |
| `claude` | `CLAUDE.md` | Claude Code |
| `cursor` | `.cursor/rules/kforge.mdc` | Cursor |
| `generic` | `AGENT.md` | Any local LLM agent or editor assistant |

## Print A Template

```bash
kforge agent print --template claude
```

Printing is useful when you want to paste the rules into another tool or review
the instruction text before installing it.

## Install A Template

```bash
kforge agent install ./my-topic --template claude
kforge agent install ./my-topic --template cursor
```

If the target file already exists, `kforge` refuses to overwrite it:

```bash
kforge agent install ./my-topic --template claude --force
```

Use `--force` only when you want to replace the existing instruction file with
the current built-in template.

## What The Templates Tell Agents

The exact wording differs by client, but every template reinforces the same
repo contract:

- preserve `raw/` as source evidence
- use `kforge context`, `kforge workflow`, `kforge search`, and `kforge inspect`
  before broad reading or editing
- write compiled articles to `wiki/`
- write durable sourced assertions to `claims/`
- save generated answer packs and reports to `outputs/`
- stage broad or uncertain changes through `reviews/`
- use `kforge promote` before filing generated output into durable knowledge
- run `kforge refresh`, `kforge doctor`, and `kforge score` before handoff

## MCP Tools

The MCP server exposes the same template operations:

- `kforge_agent_templates`
- `kforge_agent_template_print`
- `kforge_agent_template_install`

This lets an MCP-capable agent discover and install its own repo instructions
through the same deterministic API as the CLI.
