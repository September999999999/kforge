# Code of Conduct

`kforge` is an early project for people building local-first, agent-maintained
knowledge workflows. The project should stay useful, calm, and concrete.

## Expected Behavior

- Be respectful of different experience levels and workflows.
- Critique ideas, designs, and code without personal attacks.
- Keep discussions grounded in reproducible examples and project goals.
- Respect privacy. Do not ask people to share private knowledge bases, source
  documents, API keys, or proprietary data.
- Prefer small, reviewable contributions over broad rewrites.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Sharing private data without consent.
- Pressuring maintainers or contributors for unpaid support.
- Repeatedly derailing technical discussions after maintainers ask to refocus.
- Submitting intentionally malicious code, examples, prompts, or test fixtures.

## Scope

This code of conduct applies to project spaces such as issues, pull requests,
discussions, documentation, examples, and related community channels.

## Enforcement

Maintainers may remove comments, close issues, block participants, or reject
contributions that violate this code of conduct.

If you need to report a conduct issue, contact the maintainer through the
repository owner profile. If a dedicated project contact is later published,
use that contact instead.
